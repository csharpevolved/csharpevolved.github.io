Before C# 9.0, every allocation repeated the full type name even when the declaration had just stated it — `Dictionary<string, List<int>> _index = new Dictionary<string, List<int>>()` forced the reader to parse the same information twice, and the noise compounded with every generic parameter added. Target-typed `new()` collapses that to `Dictionary<string, List<int>> _index = new()`, so the declaration stays as the single source of truth and the construction site becomes almost invisible.

The benefit scales with complexity: the longer and more generic the type, the more ceremony `new()` removes, and unlike `var` it keeps the type visible at the declaration rather than hiding it entirely. That makes it a near-zero-learning-curve habit that pays dividends on every class or record that owns any fields.
